## v1.4.2

Slash commands become reliable, and Canvas is now fully configurable from a settings panel.

### Slash commands are trustworthy

This release closes the Enter-on-complete-command regression that was wiping typed args and stranding the cursor mid-line. Combined with the v1.1.0 runtime and the v1.2.0 popup (keyboard nav, IME guard, ARIA combobox), the slash-command surface is now solid.

- Pressing Enter on a complete command (e.g. `/select 1-3`) now dispatches it.
- The intercept handler checks `parseCommand(ta.value)` first and dispatches when the textarea already holds a valid command, regardless of whether the suggest popup is visible.
- The autocomplete-on-Enter path (matching Tab/click semantics) is the fallback for partial prefixes only.
- The suggest-popup row click handler has the same guard — clicking a row for a command you've already typed no longer overwrites your args.
- `/select` toasts no longer tell you to "scroll to load" — the chat surface already shows the affordance.

### Settings panel

Every Canvas behavior is now a toggle in Lumiverse's per-extension settings UI. Open Settings → Extensions → click the **Settings** button next to Canvas. Changes apply live, no reload, and persist to `layout.json`.

10 features are individually switchable, grouped under four sections:

- **Chat & Layout** — center the chat in the visible area, remember layout across sessions, smooth transitions.
- **Second Sidebar** — master switch + 4 gated sub-features (drag-to-resize, auto-mirror, mirror compact/vertical, tab labels) + consistent icon size.
- **Behavior** — auto-cleanup on uninstall.
- **Debug** — one combined mode replaces the old `localStorage.sidebarUxDebug` shim. The escape hatch is `window.__canvasDebug()`.

The second-sidebar master switch visually gates four sub-features without erasing their stored values — flip it back on and your sub-toggles are exactly where you left them.

### Patches

- **Toggle switches can now be flipped both ways.** The panel's CSS toggle buttons previously closed over the build-time initial value in their click handler, so a setting that started ON could be turned OFF but never back ON (and vice versa). The handler now reads the live state from `aria-checked`, which `refresh()` keeps in sync. Affects all 10 toggles in the panel.

> **Rebase risk:** `buildToggleControl` is unchanged on `feat/per-extension-settings-panel`, which still has the original `onChange(!value)` line. The decomp branch is a pure refactor (same signature) so the patch should apply cleanly, but **if the decomp lands first, re-apply this fix before tagging v1.4.2** or the bug returns.

### Plumbing

- `CanvasSettings` interface and `mergeCanvasSettings` helper in `src/types.ts`. Backward-compatible: layouts written by older Canvas versions hydrate against the new settings defaults without losing saved widths, tab assignments, or open/closed state.
- Settings ride on the existing `LayoutState` blob and persist via the same `SAVE_LAYOUT` IPC — no new storage key, no new permissions on `spindle.json`.
- Live update path: every setting has a corresponding handler in `applySettings()` so changes apply without a reload. The most visible one — the second-sidebar master toggle — mounts/unmounts the wrapper in place and restores the saved tab assignments.
- `dist/frontend.js` is 74.5 KB.

### Installation

Canvas installs through Lumiverse's Spindle loader from `data/extensions/canvas/repo/`. To build from source:

```bash
bash build.sh    # bundles + copies to ~/Lumiverse/data/extensions/canvas/repo/
```
