## v1.4.3

Hotfix: settings panel toggles can now be flipped in both directions.

### Fixed

- **Settings panel toggles are now fully bidirectional.** The click
  handler on each CSS toggle button captured the build-time initial
  value, so a setting that started ON could be turned OFF but never
  back ON (and vice versa). The handler now reads the live state from
  the button's `aria-checked` attribute, which the panel's refresh
  closure keeps in sync with `_settings`. Affects all 10 toggles in
  the panel.

This is a one-line fix in `buildToggleControl` (src/frontend.ts). The
complete v1.4.2 feature set is unchanged; see `.release-notes-v1.4.2.md`
for the v1.4.2 notes.

> **Rebase risk:** `buildToggleControl` is unchanged on
> `feat/per-extension-settings-panel`, which still has the original
> `onChange(!value)` line. If the decomp lands before v1.4.3 is
> installed, re-apply the fix or the bug returns.

### Installation

```bash
bash build.sh    # bundles + copies to ~/Lumiverse/data/extensions/canvas/repo/
```
